# Fate Scent ✨  
### 사주(오행) 기반 초개인화 향수 추천 프로젝트 (MVP)

---

## 1. 프로젝트 한 줄 소개
**Fate Scent**는 사용자의 **생년월일시(사주 정보)**를 바탕으로  
부족한 오행(목·화·토·금·수)을 보완해 줄 향수를 추천하는 웹 서비스입니다.

> 단순히 “인기 향수”를 추천하는 것이 아니라,  
> **왜 이 향수가 나에게 맞는지**를 설명해주는 추천 시스템을 목표로 했습니다.

---

## 2. 프로젝트를 만든 이유 (문제의식)
향수 추천은 보통 아래 방식으로 이루어집니다.

- 인기순 추천
- 많이 팔린 제품 추천
- 비슷한 사람들이 고른 향수 추천

하지만 이런 방식은 **처음 들어온 사용자(신규 유저)**에게는 약점이 있습니다.  
(= 클릭/구매 이력이 없어서 추천이 어려움, **Cold Start 문제**)

그래서 이 프로젝트에서는  
**사주(오행)라는 도메인 지식**을 활용해  
처음 들어온 사용자도 개인화 추천을 받을 수 있도록 설계했습니다.

---

## 3. 프로젝트 핵심 아이디어
### ✅ 핵심 개념
- 사용자 생년월일시 → 오행 분석 (목/화/토/금/수)
- 향수 노트/향조 → 오행 속성으로 변환
- 부족한 오행을 채워주는 향수 추천

즉,  
**“사주 기반 사용자 프로파일링 + 향수 속성 매칭”** 구조입니다.

---

## 4. MVP에서 구현한 기능
### 🔮 사용자 입력
- 이름(닉네임)
- 성별
- 생년월일
- 태어난 시간(드롭다운)
- 좋아하는 향 / 피하고 싶은 향 (선택)

### 📊 분석 결과
- 오행 분포 레이더 차트
- 강한 오행 / 부족한 오행 표시
- 부족한 오행을 보완해줄 향수 Top N 추천
- 추천 이유 설명 (어떤 향 노트가 어떤 오행을 보완하는지)

### 📝 추가 기능
- 추천 결과 CSV 저장 (로그)
- 향후 설문 데이터와 연결해 성능 검증 가능하도록 확장 구조 설계

## 5. 추천 방식 (쉽게 설명)
이 프로젝트는 크게 3단계로 추천합니다.

### 1) 사용자 오행 분석
생년월일시를 바탕으로 계절/오행 밸런스를 계산해  
**부족한 오행**과 **피해야 할 오행**을 찾습니다.

### 2) 향수 데이터 점수화
향수의 `향조(main accords)`와 `노트(notes)`를 보고  
각 향수를 오행 점수로 바꿉니다.

예시:
- Citrus / Spicy → 화(火)
- Woody / Musk → 토(土)
- Marine / Aquatic → 수(水)

### 3) 최종 추천 점수 계산
아래 요소를 합쳐 최종 점수를 만듭니다.
- 부족한 오행 보완 점수 (+)
- 피해야 할 오행 감점 (-)
- 좋아하는 향 가점 (+)
- 싫어하는 향 감점 (-)

---

## 6. 이 프로젝트의 차별점
### 1) Cold Start 해결 시도
행동 데이터가 없는 신규 사용자도  
생년월일시 정보만으로 추천 가능

### 2) 설명 가능한 추천 (Explainable Recommendation)
“왜 추천했는지”를 텍스트로 설명  
→ 사용자 설득력/만족도 향상 기대

### 3) 감성 도메인 + 데이터 분석 결합
향수처럼 취향이 강한 영역에  
도메인 지식(사주)을 결합한 하이브리드 방식
